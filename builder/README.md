# `builder/` · 【模块一】服务端工作目录构建 (Server Workspace Builder)

MC-SRE 系统的入口级核心功能，专注完成服务器文件生命周期的"从 0 到 1"。接收多种用户输入源，全自动进行依赖补全、客户端模组剥离、配置归并，最终在宿主机上编译生成"完全开箱即用、依赖完整"的服务端工作目录。

## 流水线总览（A/B 模式分流）

```
               ┌──> 模式 A: 引导包 ──> 提取 JSON ──> [多源云端下载] ──┐
[用户上传 ZIP]─┤                                                     ├──> [合并资产 & EULA] ──> 交付标准服务端目录
               └──> 模式 B: 客户端 ──> 扫描目录  ──> [本地剥离清洗] ──┘
```

| 阶段 | 子目录 | 职责 |
|------|--------|------|
| 一 | `unpack/` | 多态解包与资产校验（Zip Slip 防御 + 模式判别 + 根目录定位） |
| 二 | `metadata/` | 元数据提取（仅引导包：版本 / 加载器 / ProjectID-FileID） |
| 三 | `fetcher/` | 云端多源依赖解析与下载（PCL/HMCL 风格 + BMCLAPI 镜像） |
| 四 | `merge/` | 资产合并与配置组装（客户端黑名单清洗 + EULA 闸） |
| 五 | `deliver/` | 工作目录编译与交付（完整性校验 + 移入 server_pool） |

## 入口

- `pipeline.py`：流水线总调度，编排上述 5 个阶段，根据解包判别结果动态选择 A/B 分支。

## 输入输出契约

- **输入**：符合标准契约的 ZIP 压缩包（模式 A 引导包 / 模式 B 厚重客户端）。模式 C（自然语言）本阶段待定。
- **输出**：结构合规、依赖完整、处于 Ready-to-Run 状态的服务端工作目录（含核心 .jar、过滤后 mods/、合并后 config/、eula=true）。

## 关键边界（见各子目录 README）

- 嵌套文件夹自适应（一级单目录自动向下探查）
- EULA 拒绝即熔断、超时即终止
- 全网断流 Mod 记入 `missing_mods.log` 跳过，不卡死流水线
