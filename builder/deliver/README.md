# `builder/deliver/` · 阶段五：工作目录编译与交付

对处理完毕的临时沙箱目录进行最终完整性校验，通过后移入持久化存储区域并向下游返回工作目录信息。

## 文件职责

| 文件 | 职责 |
|------|------|
| `integrity.py` | **资产封账**：完整性校验（确保 Java 核心存在、Mod 文件无损、EULA 已签署） |
| `publisher.py` | **目录交付**：将完全态服务端文件夹移动到 `server_pool/instance_*/`，返回绝对路径、文件大小、模组总数报告 |

## 上游/下游

- 上游：`merge/` 处理完毕的临时沙箱目录
- 下游：返回工作目录绝对路径给编排层（`core/orchestrator.py`），供后续 observer 监控、brain 自愈使用
