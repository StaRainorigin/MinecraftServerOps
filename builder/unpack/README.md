# `builder/unpack/` · 阶段一：多态解包与资产校验

将用户上传的 ZIP 安全解压至系统临时沙箱目录，并自动判别输入模式（引导包 / 完整客户端）。

## 职责

| 文件 | 职责 |
|------|------|
| `safe_extract.py` | 安全解压，**必须防御 Zip Slip（目录穿越）漏洞**，禁止写出沙箱根之外 |
| `discriminator.py` | 输入模式自动判别：根目录存在 `manifest.json`/`modrinth.index.json` → 引导包；存在 `mods/` + 大量 `.jar` → 完整客户端 |
| `root_locator.py` | **畸形文件树容错**：一级目录只有一个文件夹时，深度优先向下探查定位真正的游戏根目录 |

## 边界策略

- **Zip Slip 防御**：解压前校验每个 entry 的规范化绝对路径必须以沙箱根为前缀，否则拒绝。
- **嵌套包裹自适应**：玩家压缩时常多包一层"我的整合包"文件夹，校验引擎必须 DFS 探查，严禁直接抛"无效整合包"。
- **判别结果分流**：引导包 → 进入阶段二/三（云端解析）；完整客户端 → 跳过阶段二/三，直接进 `merge/` 的客户端本地清洗。

## 上游/下游

- 上游：`pipeline.py` 传入 ZIP 路径
- 下游：`metadata/`（引导包）或直接 `merge/`（客户端）
