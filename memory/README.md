# `memory/` · 【模块四】系统记忆与自演进

MC-SRE 系统的"长期记忆中枢"与"持续学习引擎"。通过轻量级向量数据库（RAG 架构）将成功自愈经验沉淀为"肌肉记忆"，实现同类报错的**毫秒级无脑秒修**；同时通过 ChatOps 战报提供透明度，最终压降 MTTR。

## 双子系统

| 子系统 | 子目录 | 职责 |
|--------|--------|------|
| 内部 RAG 记忆流水线 | `store/` + `rag/` | 特征清洗→向量化→KV 绑定存储；故障时相似度检索短路执行 |
| 外部 ChatOps 交互管道 | `chatops/` | 战报渲染 + RLHF 反馈 |

## 三个阶段

| 阶段 | 子目录 | 职责 |
|------|--------|------|
| 一 | `store/` | 特征清洗与记忆写入（脱敏 + Embedding + 向量-ActionPlan KV 绑定） |
| 二 | `rag/` | 故障前置拦截 / 快速通道（余弦相似度 ≥0.95 短路执行，<200ms） |
| 三 | `chatops/` | 人机协同与战报生成（Markdown 战报 + RLHF 👍/👎） |

## 输入输出

- **输入**：①verify 阶段判"成功修复"的完整上下文（报错特征日志 + 最终 `ActionPlan`）②Critic 评分（需 `>= 85` 才允许写入记忆）
- **输出**：①固化到向量库的经验快照（Embeddings）②推送 ChatOps 的事故复盘战报

## 关键边界

- **经验污染**：RAG 命中但执行后仍崩 → 删除/降权该记忆，降级走 LLM 重新诊断
- **存储溢出**：LRU 淘汰 / TTL 清理，只保留高频核心经验
- **NFR**：RAG 快速通道检索 `< 200ms`（对比 LLM 推理 10s-30s）
