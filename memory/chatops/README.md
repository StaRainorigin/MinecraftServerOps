# `memory/chatops/` · 阶段三：人机协同与战报生成

无论自愈成功还是熔断失败，都将状态机流转浓缩为人类可读的 Markdown 战报推送至 IM 终端，并通过 RLHF 反馈闭环。

## 文件职责

| 文件 | 职责 |
|------|------|
| `postmortem.py` | **战报渲染**：浓缩状态机流转为 Markdown，必须列出 `[宕机时间]`、`[恢复耗时 MTTR]`、`[故障根因摘要]`、`[执行动作列表]`（精确到文件/参数）、`[当前系统状态]`（TPS/内存） |
| `notifier.py` | 推送至 ChatOps 终端（Discord / QQ / 钉钉），卡片底部附带反馈按钮 |
| `feedback.py` | **RLHF 交互**：`[👍 修得好]` 留存记忆；`[👎 乱修，回滚！]` 立即执行快照回滚（调 `brain/rollback/`）+ 向量库标记该记忆"有毒（Poisoned）" |

## 战报透出要素（来自 PRD）

- `[宕机时间]` 与 `[恢复耗时 MTTR]`（如 2 分 15 秒）
- `[故障根因摘要]`（大模型一句话总结）
- `[执行动作列表]`（精确到哪个目录哪个文件、改了哪行参数）
- `[当前系统状态]`（恢复后实时 TPS 与内存）
