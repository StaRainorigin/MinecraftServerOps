# MC-SRE · Minecraft 智能运维大脑

一套面向 Minecraft 服务端的无人值守智能运维系统。本仓库按 PRD 的四大核心功能划分顶层包，形成一个"构建 → 感知 → 自愈 → 记忆"的完整闭环。

## 四大模块闭环

```
┌─────────────────────────────────────────────────────────────────────┐
│                      MC-SRE 系统总览                                │
│                                                                     │
│  [模块一 builder]          [模块二 observer]                        │
│   工作目录构建      ──▶ ①运行─▶  全天候可观测性                     │
│   (从 0 到 1)                      │ 异常事件 AnomalyEvent          │
│        │                          ▼                                 │
│        │                  [模块三 brain]                            │
│        │                    AI 决策大脑 + 自愈闭环                   │
│        │                          │ 成功修复上下文                   │
│        │                          ▼                                 │
│        │                  [模块四 memory]                           │
│        │                    系统记忆与自演进 (RAG + ChatOps)         │
│        │                          │ 命中记忆 → 秒级短路修复          │
│        └──────────────────────────┘                                 │
└─────────────────────────────────────────────────────────────────────┘
```

| 模块 | 包名 | 定位 | PRD 对应 |
|------|------|------|----------|
| 一 | `builder/` | 服务端工作目录构建 (Server Workspace Builder) | 多模态输入 → 开箱即用目录 |
| 二 | `observer/` | 全天候可观测性与异常感知 (低开销守护进程) | 日志/指标 → AnomalyEvent |
| 三 | `brain/` | AI 决策大脑与自愈闭环 (LangGraph 图状态机) | 根因诊断 → 安全修复 → 反思 |
| 四 | `memory/` | 系统记忆与自演进 (RAG + ChatOps) | 经验沉淀 → 毫秒级秒修 |

## 目录约定

- 每个目录下的 `README.md` 说明该目录的职责，并标注对应的 PRD 模块/阶段编号。
- `.py` 文件目前均为**占位骨架**（仅模块 docstring），逻辑待后续按模块填充。
- `core/` 为跨模块共享内核（契约、路径常量、事件总线、顶层编排）。
- `infra/` 为外部依赖适配层（Docker / RCON / LLM / IM 平台）。

> 本阶段仅生成源码树与说明占位，不包含构建配置、依赖清单与测试。
